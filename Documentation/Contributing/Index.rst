============
Contributing
============

Contributions to the TWOH Kickstarter extension are welcome!

.. _contributing-how:

How to Contribute
=================

1. **Fork the repository** on GitHub
2. **Create a feature branch** from the main branch
3. **Make your changes** and commit them with descriptive messages
4. **Push your changes** to your fork
5. **Create a Pull Request** against the main repository

.. _contributing-guidelines:

Guidelines
==========

Code Style
----------

*  Follow the `TYPO3 Coding Guidelines <https://docs.typo3.org/m/typo3/reference-coreapi/main/en-us/CodingGuidelines/Index.html>`__
*  Use PSR-12 coding standard for PHP
*  Write meaningful commit messages

Testing
-------

*  Ensure your changes don't break existing functionality
*  Test with the supported PHP versions (8.2, 8.3)
*  Test with TYPO3 13.4

Documentation
-------------

*  Update documentation when adding new features
*  Use reStructuredText format
*  Follow the `TYPO3 Documentation Guidelines <https://docs.typo3.org/m/typo3/docs-how-to-document/main/en-us/>`__

.. _contributing-issues:

Reporting Issues
================

Found a bug? Please report it on GitHub:

*  **Issue Tracker:** `github.com/twohreichel/twoh_kickstarter/issues <https://github.com/twohreichel/twoh_kickstarter/issues>`__

When reporting issues, please include:

*  TYPO3 version
*  PHP version
*  Extension version
*  Steps to reproduce the issue
*  Expected behavior
*  Actual behavior

.. _contributing-feature-requests:

Feature Requests
================

Have an idea for a new feature? We'd love to hear it!

1. Check if the feature has already been requested
2. Open a new issue describing the feature
3. Explain why the feature would be useful

.. _contributing-pull-requests:

Pull Requests
=============

When creating a Pull Request:

*  Reference related issues in the PR description
*  Provide a clear description of the changes
*  Update the CHANGELOG if applicable
*  Ensure all tests pass
